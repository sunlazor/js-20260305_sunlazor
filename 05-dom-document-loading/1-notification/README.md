# NotificationMessage

В этом задании необходимо реализовать компонент `NotificationMessage` — всплывающее уведомление, которое появляется на странице и автоматически исчезает через заданное время.

Такие уведомления часто используются для информирования пользователя об успешном сохранении данных ("success") или об ошибках ("error").

![Notification](notification.gif)

## Описание задачи

Вам нужно написать класс на TypeScript, который создает DOM-элемент уведомления и управляет его жизненным циклом (показ, скрытие, таймер).

Ключевая особенность: **На странице может быть только одно уведомление одновременно.** Если показывается новое уведомление, старое должно быть удалено.

## Требования к реализации

Работа ведется в файле `src/index.ts`.

### 1. Интерфейс класса

```typescript
new NotificationMessage(message, options)
```

*   **message**: строка с текстом сообщения.
*   **options**: объект с настройками (опционально):
    *   `duration`: длительность показа в миллисекундах (по умолчанию 2000ms).
    *   `type`: тип уведомления, `'success'` или `'error'` (по умолчанию 'success'). Влияет на CSS-класс.

### 2. Методы

*   `show(target)`: отображает уведомление на странице.
    *   `target`: DOM-элемент, в который нужно добавить уведомление. Если не передан, то добавляем в `document.body`.
    *   При вызове `show` должен запускаться таймер, который удалит уведомление через `duration` миллисекунд.
*   `remove()`: удаляет элемент уведомления из DOM.
*   `destroy()`: полностью удаляет компонент (удаляет элемент из DOM, очищает таймеры).

### 3. Поведение (Singleton-like)

Чтобы выполнить требование "только одно уведомление на странице", вам пригодится сохранить ссылку на *текущий активный экземпляр* уведомления.

Когда создается или показывается новое уведомление:
1.  Проверьте, есть ли уже активное уведомление.
2.  Если есть — удалите его.
3.  Сохраните ссылку на новое уведомление как активное.

## Подсказка

Обратите внимание на CSS-переменную `--value` в атрибуте `style`. Она используется для анимации полоски таймера (значение должно быть в секундах, например `2s`).

### Статическое свойство
Для хранения ссылки на активное уведомление удобно использовать статическое свойство класса.

```typescript
export default class NotificationMessage {
  static activeNotification: NotificationMessage;

  constructor(...) {
    if (NotificationMessage.activeNotification) {
      NotificationMessage.activeNotification.remove();
    }
    // ...
    NotificationMessage.activeNotification = this;
  }
}
```
Это позволит вам иметь доступ к текущему уведомлению из любого нового экземпляра класса.
